# pyo3-ffi

Raw Rust FFI bindings to the Python C API (Python/C API Reference Manual). This crate is
hand-maintained — not generated by bindgen — and mirrors the structure of CPython's
`Include/` and `Include/cpython/` header trees.

## Purpose

Provides low-level, unsafe `extern "C"` declarations, struct definitions, and helper
macros matching the C API exactly. Regular PyO3 users should never import this crate
directly; the `pyo3` crate re-exports everything at a safe, ergonomic level.

## Supported runtimes and versions

- CPython 3.8 through 3.15 (inclusive)
- PyPy 7.3 (Python 3.11+)
- GraalPy 24.0+ (Python 3.10+)

## Conditional compilation model

Three orthogonal cfg axes are used throughout the crate:

| cfg flag | Meaning |
|---|---|
| `Py_3_8` … `Py_3_15` | Minimum Python version required for the annotated item |
| `Py_LIMITED_API` | ABI3 stable-ABI subset (enabled by the `abi3` feature) |
| `PyPy` / `GraalPy` | Alternative runtime; excludes CPython-only APIs |
| `Py_GIL_DISABLED` | Free-threaded CPython build |

The flags are set by `pyo3-build-config` at build time via `use_pyo3_cfgs()` in
`build.rs`. Downstream crates that need to query these flags must add
`pyo3-build-config` as a build dependency.

## Key files

- `Cargo.toml` — declares `links = "python"` (one Python link per build graph);
  `abi3-py38` … `abi3-py315` chained features for minimum-version selection.
- `build.rs` — delegates to `pyo3-build-config` for Python interpreter detection,
  cfg emission, and import-lib generation on Windows.
- `src/` — all FFI source; see `src/CLAUDE.md`.

## Relationship to other crates

- `pyo3` — the safe wrapper crate; depends on `pyo3-ffi` and re-exports types.
- `pyo3-build-config` — build dependency that detects Python and emits cfg flags.
- `pyo3-macros` / `pyo3-macros-backend` — proc-macro crates that generate code
  using `pyo3`'s safe layer, not `pyo3-ffi` directly.
