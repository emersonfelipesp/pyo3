# PyO3 v0.28.3 — Vendor Reference

PyO3 is a Rust framework for creating Python extension modules and embedding
the Python interpreter in Rust programs. It wraps CPython's C API behind a
safe Rust interface. This checkout is a vendor reference clone of
[PyO3/pyo3](https://github.com/PyO3/pyo3) pinned at v0.28.3; it is not
actively developed here.

## 6-Layer Architecture

| Layer | Crate / Path | Role |
|---|---|---|
| FFI bindings | `pyo3-ffi/` | Raw `unsafe` CPython C API declarations |
| Build config | `pyo3-build-config/` | Detects Python version, sets `cfg` flags |
| Object bindings | `src/types/` | Safe Rust types wrapping Python built-ins |
| PyClass system | `src/pyclass/`, `src/pycell/`, `src/impl_/` | `#[pyclass]` macro runtime support |
| Proc-macros | `pyo3-macros/`, `pyo3-macros-backend/` | `#[pyclass]`, `#[pyfunction]`, `#[pymodule]` |
| Introspection | `pyo3-introspection/` | Linker-section-based metadata for type stubs |

## Crate Dependency Graph

```
pyo3  ──depends on──▶  pyo3-ffi        (FFI layer, always)
pyo3  ──depends on──▶  pyo3-macros     (optional, feature = "macros")
pyo3-macros  ──▶  pyo3-macros-backend
pyo3-ffi  ──▶  pyo3-build-config      (build.rs only)
pyo3  ──▶  pyo3-build-config           (build.rs only, forwarded via `links`)
```

The `links = "pyo3-python"` field in `Cargo.toml` lets downstream crates
receive build metadata from `pyo3-build-config` without depending on it
directly.

## Python Compatibility

- CPython 3.8–3.14 (3.15 in progress)
- PyPy 7.3 (Python 3.11+)
- GraalPy 24.0+ (Python 3.10+)
- Free-threaded CPython (`Py_GIL_DISABLED`): supported via `nightly` feature
- ABI3 / Limited API: supported via `abi3` feature (forward-compatible with
  future Python versions per PEP 384)

## Key Environment Variables

| Variable | Effect |
|---|---|
| `PYO3_CONFIG_FILE` | Path to a hand-written build config; skips Python detection |
| `PYO3_NO_PYTHON` | Disables all Python interpreter detection at build time |
| `PYO3_CROSS_LIB_DIR` | Cross-compile: directory containing the Python library |
| `PYO3_CROSS_PYTHON_VERSION` | Cross-compile: target Python version |
| `PYO3_CROSS_PYTHON_IMPLEMENTATION` | Cross-compile: `CPython`, `PyPy`, or `GraalPy` |

## `rustc` cfg Flags Set by pyo3-build-config

`Py_3_8` through `Py_3_14`, `Py_LIMITED_API`, `Py_GIL_DISABLED`, `PyPy`,
`GraalPy`. Gate version-specific code with `#[cfg(Py_3_12)]` etc.

## How to Read the Repo

1. Start with `src/lib.rs` for the public API surface and feature flag docs.
2. `src/instance.rs` (100K) — `Py<T>` and `Bound<'py, T>` smart pointers.
3. `src/marker.rs` (36K) — `Python<'py>` GIL token.
4. `src/types/` — safe wrappers around Python built-in types.
5. `src/impl_/` — hidden runtime support generated by proc-macros.
6. `pyo3-macros-backend/` — proc-macro implementation (readable Rust).
