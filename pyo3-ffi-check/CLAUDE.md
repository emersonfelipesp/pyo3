# pyo3-ffi-check

CI validation tool that cross-checks `pyo3-ffi`'s hand-written FFI struct definitions against bindings generated by `bindgen` from the actual CPython C headers. Not published to crates.io.

## Purpose

Catches size or alignment mismatches in `pyo3-ffi` before they reach users. When a Python version updates its internal C structs, this tool flags the difference. It exits nonzero if any struct size or field offset differs.

## Sub-crate Structure

This is its own Cargo workspace with three members:

| Crate | Path | Role |
|---|---|---|
| `pyo3-ffi-check` | `src/main.rs` | Comparison driver: iterates structs, reports mismatches |
| `pyo3-ffi-check-macro` | `macro/` | Proc-macro `for_all_fields!` — generates field-offset checks for each struct field |
| `pyo3-ffi-check-definitions` | `definitions/` | Bindgen-generated bindings; `build.rs` runs bindgen against `wrapper.h` |

## How It Works

1. `definitions/build.rs` runs `bindgen` on `definitions/wrapper.h` (which includes CPython headers) to produce `bindings.rs`.
2. `src/main.rs` compares `std::mem::size_of::<pyo3_ffi::StructName>()` against `std::mem::size_of::<bindings::StructName>()` for each registered struct.
3. The `for_all_fields!` proc-macro checks field-level offsets via `std::mem::offset_of!`.
4. Any mismatch is printed and the process exits with a nonzero code.

## Key Files

- `src/main.rs` (3.7K) — main comparison loop with `check_struct!` macro
- `macro/src/` — proc-macro implementation for `for_all_fields!`
- `definitions/build.rs` (3.4K) — bindgen invocation with appropriate Python include paths
- `definitions/wrapper.h` — minimal C header that includes the CPython headers to bind against
- `build.rs` (1.8K) — top-level build script forwarding Python config from `pyo3-build-config`
- `README.md` — one-paragraph summary of purpose

## Relationship to pyo3-ffi

- Depends on `pyo3-ffi` (from workspace path) and on `pyo3-build-config` for Python version detection.
- Run as part of PyO3's CI matrix against each supported CPython version.
- Not run during normal `cargo test` of the main workspace.

## Running

```bash
cd pyo3-ffi-check
cargo run
```
